| 2026-05-27 | **`_RouterSwipeGesture` fix — recognizer attaches to `UIWindow`, not to an overlay view.** Luis: "the hand gesture doesnt seem to work." Root cause: the prior `PassthroughView.hitTest` returning nil to let single touches pass through to SwiftUI also prevented the overlay view from receiving touches at ALL, so the UIPanGestureRecognizer attached to it never fired. Classic UIKit gotcha: hitTest nil = no touches = no gestures.<br>**Fix:** rewrote to attach the pan recognizer to the key UIWindow directly. The window receives every touch by default; `cancelsTouchesInView = false` + `shouldRecognizeSimultaneouslyWith` returning true keeps SwiftUI's gestures (taps, scrolls, drags) working unaffected. Idempotent attach (re-runs DispatchQueue.main.async until window resolvable on first launch); detach via `dismantleUIView`. Threshold + dominance ratio unchanged (80pt min, 2× X-over-Y).<br>Audit clean. Build clean. | 🦊 Designer + Luis |
