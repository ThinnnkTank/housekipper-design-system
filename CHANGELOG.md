| 2026-05-23 | **DsBadge locked ✅.** Universal severity counter overlay. Severity-driven content: `.count(N)` for attention (numeric, capped at "99+"), `.urgent` for the "!!" symbolic alert, healthy renders nothing. Single `Capsule` shape adapts to content — single character renders circular, multi-character extends into a pill. Render-only Primitive; positioning lives in the consumer Component via `.overlay` + shape-specific `Inventory.badgeOverhang{Rect,Pill}`. New tokens: `Font.hkBadge` (DM Sans Bold 13pt — single role for both modes, intentional step out of mono-for-utility), `InventoryToken.badgePaddingH` (6pt internal), `InventoryToken.badgeOverhangPill` (2pt). Bumped: `badgeSize` 18→20, `badgeSizeSmall` 15→17, `badgeBorderWidth` 1.5→2 (matches iOS-native badge ring weight). | 🦊 Designer + Luis |
| 2026-05-23 | **Snapping rule refined.** Foundations now distinguishes macro from micro. Sub-grid values (1, 1.5, 2pt) are valid currency *inside* a primitive — badge rings, hairline strokes, small offsets where rounded geometry demands precision. They remain forbidden for spacing/layout between components. The carve-out is the primitive interior, not the surrounding gaps. | 🦊 Designer + Luis |
| 2026-05-23 | **Commit hygiene: no Co-Authored-By trailers.** Removed the trailer requirement from CLAUDE.md commit-message-style. Applies to every agent. | 🦊 Designer + Luis |
| 2026-05-23 | **DsKeyButton locked ✅.** Severity-driven tile button for rooms / outdoor / systems. Two shapes: `.rect` (Radius.sm) for rooms+outdoor, `.pill` (Capsule) for systems — corner-shape is the only visual difference; size, padding, type, and behavior are uniform (`Inventory.tileMinWidth × tileHeight` = 100×60pt flex). Severity ladder: healthy (1pt ink border, clear fill), attention (2pt signal border, signal icon, ink label, clear fill), urgent (2pt signal border + `signalTint` fill). Press strategy: **invert** — pressed flips to ink fill + paper content (new vocabulary distinct from DsButton's `soften`). Asymmetric press: instant down, `Motion.standard` (300ms — see below) on release. Hit area uses `.contentShape(...)` matching tile shape — required because `.healthy` clear fill otherwise leaves dead corners. New tokens: `InventoryToken` BaseToken category (tile + rail + badge geometry), `Inventory` SemanticToken pass-through. | 🦊 Designer + Luis |
| 2026-05-23 | **Motion.standard 220 → 300ms.** Bumped `MotionToken.dBase` from 0.220 to 0.300. Drives the asymmetric press release across DsButton + DsKeyButton (every Primitive using soften or invert press). 220ms felt twitchy on iPad. | 🦊 Designer + Luis |
| 2026-05-23 | **Snapping rule codified.** Foundations now requires every token tweak to snap to an existing stop (17→20, 14→10, 18→20). Adding a new stop is a "decisions that must be surfaced" moment. Applies to every token system (spacing, radius, opacity, font size, border width, motion duration). | 🦊 Designer + Luis |
| 2026-05-22 | **DsButton refinements + locked ✅.** Multiple iterations on iPad device:<br>· Type bump: 13/12/11pt (large/small/micro). Micro uses DM Sans Bold (DM Mono ships no Bold).<br>· Border: 2pt → 1pt (was reading too heavy).<br>· Icon weight: decoupled from label via `IconWeight.action = .bold` so stroke is uniform across sizes.<br>· Tracking by size: large `trackingLabel` (+1.0), small `trackingSnug` (+0.4, new), micro `trackingMicro` (+1.1, new).<br>· Urgent-disabled fill: drifted to `signalTint` (12%) — corrected to new `signalMuted` (40%) for confident salmon presence. Foreground flipped to `paper` for legibility.<br>· Secondary-disabled border: `ink40` (50%) → `ink20` (22%) — also affects primary/secondary press feedback per the rule.<br>· Press animation: asymmetric.
